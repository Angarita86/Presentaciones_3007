Manual de usuario

Guía rápida para actualizar y compartir este informe interactivo.

1. ¿Qué es este archivo?

Es un informe interactivo autocontenido: un solo archivo HTML que se abre en cualquier navegador (Chrome, Edge, Firefox) haciendo doble clic. No requiere instalar nada. Solo necesita internet la primera vez que se abre, para descargar dos librerías de visualización.

2. Actualizar los datos (Excel)

En la hoja Configuración → 1 · Cargar Excel del corte, selecciona el archivo .xlsx del nuevo corte. Las tablas del Informe y el Mapa de Colombia se recalculan al instante. Estructura esperada del Excel — los nombres de hoja deben coincidir exactamente:

  • Maquinaria_Amarilla: ID, Caso Nombre, Medida, Producto / Medida, Accion, Actividad / Accion, Proyeccion de costos, Observaciones
  • Documentacion_Completa: Tipo, Caso Nombre, Medida ID, Producto / Medida, Accion ID, Actividad / Accion, Proyeccion de costos
  • Acompanamiento_Documentacion: Tipo, Caso Nombre, Macroregion, Caso DT, Caso Mpio, Caso Dpto
  • Acciones_Rehabilitacion: Tipo, Caso Nombre, Medida ID, Producto / Medida, Accion ID, Actividad / Accion, Proyeccion de costos
  • Seguimiento_Contactabilidad: Zona, Sin Respuesta, Con Respuesta
  • Insumo_Sujetos (alimenta el Mapa Colombia): TIPO SUJETO, ID sujeto, Sujeto, ZONA, DANE, DEPARTAMENTO, MUNCIPIO, Accion, estado de implementación, Fecha, Descripción de la acción, OBSERVACIONES

Puedes cargar hojas parciales: las que falten simplemente no se actualizan y mantienen su último dato.

3. Fecha de corte, logo y nombres de las hojas

Desde la hoja Configuración también puedes: cambiar la fecha de corte (se refleja en la barra superior y en la portada); cargar un logo en imagen (reemplaza los logos de texto de la esquina superior izquierda y aparece en la portada, sin afectar la fecha de corte); y renombrar las pestañas "Informe" y "Registro Fotográfico" si se requiere otro título.

4. Registro fotográfico

En Configuración → 5 · Cargar registro fotográfico puedes seleccionar varias imágenes o una carpeta completa. Las fotos se agrupan automáticamente por Sujeto usando el nombre del archivo: por ejemplo, Flamenco_1.jpg y Flamenco_2.jpg forman el grupo "Flamenco". Cada carga reemplaza el registro anterior. En la hoja "Registro Fotográfico", haz clic en cualquier miniatura para verla ampliada (cierra con la X, clic afuera o tecla Esc).

5. Mapa de implementación

El mapa colorea los departamentos según el número de acciones registradas en la hoja Insumo_Sujetos del Excel, sin importar su estado de implementación. Pasa el mouse sobre un departamento para ver sus acciones, sujetos y municipios; el panel derecho lista los departamentos ordenados de mayor a menor actividad.

6. Diseño de hojas (hoja Diapositivas del Excel)

El orden, la visibilidad y el contenido de las secciones del Informe se definen en la hoja Diapositivas del Excel. Puedes editarla directamente en Excel, o de forma visual en Configuración → Diseño de hojas: activar/desactivar secciones, reordenarlas (▲▼), y crear secciones tipo "bloques" (de 1 a 4 bloques horizontales, cada uno con 1 a 4 celdas de texto, imagen o imagen+texto, con título, detalle, cifra destacada, fondo de color y ajuste de imagen). Al terminar, usa "Descargar Excel actualizado" y sube ese archivo al repositorio como data/informe.xlsx para que el diseño quede permanente. Las imágenes se referencian por su nombre de archivo en la carpeta fotos/.

7. Casos Emblemáticos y Portada editable

El menú incluye la pestaña Casos Emblemáticos con hasta 5 hojas (Caso 1 a Caso 5), que se activan, renombran y arman solo con secciones de bloques desde Diseño de hojas (barra superior: Portada · Informe · Casos). La Portada también es editable ahí: cada campo de texto tiene estilo (eyebrow, título, subtítulo, etiqueta, convenio o normal), se puede ocultar, reordenar o agregar campos nuevos. Todo se guarda en la hoja Diapositivas del Excel (columna Hoja).

8. Guardar directo en GitHub (sin descargar archivos)

En Configuración → 7 · Guardar directo en el repositorio puedes pegar un token de GitHub (Fine-grained, solo este repositorio, permiso Contents: Read and write) y publicar el Excel de diseño y la configuración con un clic. Los cambios quedan visibles para todos en ~1 minuto. Sin token, sigue disponible el flujo manual: Descargar Excel / config.json y subirlos al repositorio.

9. Copia de seguridad y cómo revertir un error

Con la casilla "Crear copia de seguridad" activada (por defecto), cada vez que guardas desde la app se guarda automáticamente una copia del Excel anterior en la carpeta backups/ del repositorio, con fecha y hora en el nombre (ej. informe_2026-07-06_1453.xlsx). Para revertir un error: entra a esa carpeta en GitHub, descarga la copia que quieras recuperar y súbela reemplazando data/informe.xlsx.

Además, GitHub guarda el historial completo de todo el repositorio aunque no uses esta casilla: en la página del archivo (por ejemplo index.html o data/informe.xlsx) hay un botón "History" que muestra todas las versiones anteriores y permite descargar cualquiera de ellas.

10. Eliminar tablas o secciones

El botón "Eliminar" del editor ya no es exclusivo de las secciones de bloques: también funciona sobre las tablas (fijas o de hojas nuevas del Excel). Al eliminar una tabla, desaparece de la vista del Informe/Caso y, al descargar o guardar el Excel actualizado, esa hoja ya no aparecerá referenciada en Diapositivas — el cambio queda reflejado en ambos lados.

11. Eliminar una celda específica de un bloque

Cada celda de una sección de bloques tiene un botón ✕ en su esquina superior derecha para borrarla individualmente, sin afectar las demás celdas del mismo bloque ni tener que rehacer el bloque completo. Si es la única celda del bloque, al eliminarla se pregunta si también quieres quitar el bloque vacío.

12. Corte de julio 2026

Este corte incorporó la hoja de datos Acciones_suspendidas (nueva sección "Acciones Suspendidas" en el Informe) y el Caso Emblemático "Kankuruas — Resguardo Indígena Kankuamo", construido a partir del primer informe de avance de construcción de Kankuruas del Resguardo Indígena Kankuamo. La hoja de datos Sujetos_a_implementar quedó disponible en el Excel para uso futuro (no está referenciada en ninguna sección todavía).

13. Las imágenes no aparecen

Si subiste fotos a fotos/ o imagenes/ directamente en GitHub y no las ves en la galería o en el selector del editor, lo más probable es que falte agregarlas al manifest.json correspondiente. La forma más confiable de evitar este paso manual es usar Configuración → 8 · Imágenes disponibles en el repositorio → "Actualizar lista de imágenes desde GitHub" (con el token puesto): detecta automáticamente todas las imágenes reales de esas dos carpetas, sin depender de los manifest.json.

9. Importante al compartir

Los cambios hechos en el navegador (Excel cargado, fotos, logo, fecha) no se guardan dentro del archivo HTML: viven solo en la sesión abierta. Para presentar, abre el archivo y carga los insumos al inicio de la reunión; o comparte el HTML junto con el Excel del corte para que cada persona los cargue. Si se requiere dejar una versión "congelada" con datos actualizados de forma permanente, se debe generar una nueva versión del archivo.

Configuración

Panel de configuración

Carga el Excel del corte, actualiza el logo y la fecha de corte antes de compartir el informe.

1 · Cargar Excel del corte

Carga un archivo .xlsx con las hojas del informe (tablas) y, si quieres actualizar el mapa, la hoja Insumo_Sujetos.

Sin archivo cargado — se muestran los datos de ejemplo (corte 16-jun-2026).

2 · Fecha de corte

Esta fecha se muestra en la parte superior del informe y en la portada.

⚠️ El cambio aplica solo a esta sesión. Para dejarlo permanente, descarga el config.json (tarjeta "Guardar configuración") y súbelo al repositorio.

3 · Logo a presentar

Al cargar una imagen, reemplaza los logos de texto en la esquina superior izquierda y en la portada. La fecha de corte (a la derecha) no se modifica.

Sin logo personalizado — se muestran los logos por defecto.

4 · Cargar registro fotográfico

⚠️ Solo para la galería de la hoja "Registro Fotográfico" — reemplaza las fotos actuales de esa galería. Se agrupan por Sujeto según el nombre del archivo (Flamenco_1.jpg → grupo "Flamenco"). Permanente: carpeta fotos/ + fotos/manifest.json del repositorio.

Sin fotos cargadas — se muestran las fotos de ejemplo del proyecto.

4b · Imágenes del informe (diapositivas)

Estas imágenes no tocan el Registro Fotográfico: quedan disponibles en el selector "Imagen" del Diseño de hojas, para usarlas dentro de las diapositivas. Permanente: súbelas a la carpeta imagenes/ del repositorio y regístralas en imagenes/manifest.json.

Sin imágenes de sesión — se listan las de la carpeta imagenes/ y fotos/ del repositorio.

5 · Nombres de las hojas

Si prefieres otros nombres para las hojas del menú, cámbialos aquí — se actualizan al instante en la barra de navegación. La estructura detallada del Excel y demás instrucciones están en el Manual de usuario (menú de la izquierda).

6 · Guardar configuración en el repositorio

Los ajustes hechos aquí (fecha, nombres de hojas, logo) viven solo en esta sesión del navegador. Para dejarlos permanentes, descarga el archivo de configuración y súbelo al repositorio reemplazando el config.json existente. Si cargaste un logo, guárdalo también como assets/logo.png.

7 · Guardar directo en el repositorio (GitHub)

Con un token de GitHub publicas los cambios (diseño + configuración) sin descargar ni subir archivos. Crea el token en GitHub → Settings → Developer settings → Fine-grained tokens, con acceso solo a este repositorio y permiso Contents: Read and write. El token se recuerda en este equipo (queda guardado en el navegador hasta que uses "Olvidar token" o expire). En computadores compartidos, usa "Olvidar token" al terminar.

8 · Imágenes disponibles en el repositorio

Usa el mismo token de arriba para que el editor detecte automáticamente las imágenes que ya estén subidas a fotos/ e imagenes/ en GitHub — sin depender de mantener los manifest.json actualizados a mano.
